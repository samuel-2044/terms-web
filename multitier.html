Multitier Architecture

A system design approach that separates applications into layers.

What is Multitier Architecture?

Multitier architecture (also called n-tier architecture) is a software design model where an application is divided into multiple layers, each responsible for a specific function.

Main Tiers

  • Presentation Tier: User interface (browser or frontend)
  • Application Tier: Business logic and processing
  • Data Tier: Database and data storage

How It Works

When a user interacts with a system, requests move from the presentation tier to the application tier, and finally to the data tier. Responses travel back in reverse order.

Benefits

  • Improved scalability
  • Easier maintenance
  • Better security (layer isolation)
  • Flexible development and deployment

Example

In a web application: the browser is the presentation layer, the backend server handles logic, and a database stores user data.

💡 Multitier systems power most modern web applications today.
