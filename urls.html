Uniform Resource Locator (URL)

The address system used to locate resources on the Internet.

Definition of URL

A Uniform Resource Locator (URL) is the standardized address system used to precisely locate and access any resource on the Internet or private networks. URLs provide a consistent method for browsers and other clients to specify exactly what resource they want from where, using a defined scheme that includes protocol, domain, path, and optional parameters. Every website, image, API endpoint, and web service has a unique URL. URLs are the foundation of web navigation and were developed by Tim Berners-Lee as part of the World Wide Web in 1989. Modern URLs can be complex, containing authentication information, port numbers, query parameters, and fragments, yet the browser interprets them to execute the correct connection and retrieve the desired resource.

Complete URL Structure and Components

A full URL can contain: protocol://[user:password@]host[:port]/path?query_string#fragment

  • Protocol/Scheme: Defines the communication method (http, https, ftp, mailto, file). HTTPS is secured with encryption, while HTTP is unencrypted
  • User and Password: Optional authentication credentials for accessing protected resources (rarely used in modern URLs for security reasons)
  • Host: Domain name or IP address of the server hosting the resource
  • Port: Optional specific port on the server (default 80 for HTTP, 443 for HTTPS, 21 for FTP)
  • Path: Location of the resource on the server's file system or application (e.g., /products/electronics/laptops)
  • Query String: Parameters passed to the server as key-value pairs starting with ? and separated by & (e.g., ?search=laptop&price=500). Used by search engines, filters, and pagination
  • Fragment: Specifies a location within the page (starting with #), causing the browser to scroll to that section without requesting new content from server

URL Encoding (Percent Encoding)

URLs contain only safe ASCII characters. Special characters must be encoded as %XX where XX is the character's hexadecimal value. Spaces become %20 or +. Special characters like ? = & # must be encoded in query strings to avoid conflicts. This encoding ensures URLs remain valid and unambiguous across all systems and protocols.

Types of URLs and Their Uses

  • Absolute URLs: Complete addresses including protocol and domain; work from any location on any website. Used for external links to other domains
  • Relative URLs: Paths relative to current location; use ../ to go up one directory or paths without domain. Used for internal links within same website for portability
  • Root-Relative URLs: Start with / and reference from the website root, independent of current page location

Structure of a URL

Example:

https://www.example.com/path/to/page.html
  • https: Protocol (communication method)
  • www.example.com: Domain name
  • /path/to/page.html: Resource location (path)

Types of URLs

  • Absolute URL: Full web address including domain
  • Relative URL: Path relative to current page

Importance of URLs

  • Enable navigation on the web
  • Identify unique resources
  • Used by browsers to load web pages

URL = The "address system" of the Internet

Complete URL Anatomy

https://user:pass@www.example.com:8080/path/to/page.html?query=value#fragment

  • https: Protocol/Scheme (http, https, ftp, mailto)
  • user:pass@: Optional authentication credentials
  • www.example.com: Domain name or IP address
  • :8080: Port number (optional, default 80 for HTTP, 443 for HTTPS)
  • /path/to/page.html: Path to the resource
  • ?query=value: Query string for passing parameters
  • #fragment: Anchor to specific section within the page

URL Encoding (Percent Encoding)

Special characters must be encoded to be URL-safe:

  • Space becomes %20 or +
  • ? becomes %3F
  • = becomes %3D
  • & becomes %26
  • # becomes %23
  • Example: search?q=hello world&page=1 becomes search?q=hello%20world%26page%3D1

URL Shorteners

  • Bitly: Custom short URLs with analytics
  • TinyURL: Simple URL shortening since 2002
  • Google URL Shortener: Discontinued (was goo.gl)
  • Twitter t.co: Automatic shortening for tweets
  • Use cases: Social media, SMS, QR codes, tracking links

Best Practices for URLs

  • Keep them short and descriptive
  • Use hyphens (-) not underscores (_) to separate words
  • Avoid special characters and spaces
  • Use lowercase letters (URLs are case-sensitive)
  • Include relevant keywords for SEO
  • Use canonical URLs to avoid duplicate content
  • Implement proper 301 redirects when changing URLs
  • Use HTTPS for security and SEO benefits

Interesting URL Facts

  • The longest URL ever recorded was over 2,000 characters
  • URLs can contain emojis (encoded as UTF-8)
  • Internationalized Domain Names (IDN) support non-ASCII characters
  • Punycode converts Unicode to ASCII (xn-- prefix in URLs)
  • Some URLs use "slug" - human-readable path components

URL Security Considerations

  • Phishing: Fake URLs mimicking legitimate sites (amaz0n.com)
  • Open Redirects: Malicious redirects through trusted domains
  • URL Obfuscation: Hiding true destination with redirects
  • Always verify: Check domain spelling and use HTTPS
  • Hover before clicking: Preview URL destination
