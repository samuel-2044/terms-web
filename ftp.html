File Transfer Protocol (FTP)

A standard protocol used to transfer files between client and server systems.

What is FTP?

File Transfer Protocol (FTP) is a network protocol used to transfer files between a client and a server over a computer network. It follows a client-server model and uses separate control and data connections.

How FTP Works

  • Control Connection: Sends commands and responses (Port 21)
  • Data Connection: Transfers files (Port 20 or dynamic ports)
  • Supports active and passive modes for connection handling

History

FTP was standardized in the early 1970s and became one of the earliest methods for transferring files over the Internet. It has since evolved into more secure versions like SFTP and FTPS.

Use Cases

  • Website file uploads and management
  • Software distribution
  • Large data backups and transfers

Security

  • Standard FTP is not encrypted
  • Credentials can be intercepted if not secured
  • Use SFTP or FTPS for secure transfers

Related Protocols

  • SFTP: Secure file transfer over SSH
  • FTPS: FTP with SSL/TLS encryption
  • TFTP: Lightweight file transfer protocol

💡 FTP is still widely used, but secure versions are now preferred.

FTP Connection Modes

FTP uses different modes to establish data connections:

  • Active Mode: Server initiates data connection to client's port 20
    • Client sends PORT command with its IP and port
    • Server connects from port 20 to specified client port
    • Can fail with firewalls/NAT
  • Passive Mode: Client initiates both control and data connections
    • Client sends PASV command
    • Server responds with IP and random port number
    • Client connects to that port for data transfer
    • Better for firewalls and NAT

Common FTP Commands

  • USER: Specify username for authentication
  • PASS: Specify password
  • LIST: List files in current directory
  • RETR: Retrieve/download a file
  • STOR: Store/upload a file
  • DELE: Delete a file
  • MKD: Create a directory
  • PWD: Print working directory
  • QUIT: Close connection

FTP Response Codes

  • 1xx: Positive preliminary reply
  • 2xx: Positive completion (200 Command OK, 226 Transfer Complete)
  • 3xx: Positive intermediate (331 Username OK, need password)
  • 4xx: Transient negative (421 Service unavailable, 451 Local error)
  • 5xx: Permanent negative (550 File not found, 530 Not logged in)

SFTP vs FTPS Comparison

Feature SFTP FTPS
Protocol Base SSH (Secure Shell) FTP with SSL/TLS
Port 22 990 (implicit), 21 (explicit)
Firewall Friendly Yes - single port No - requires multiple ports
Platform Support Universal via SSH Requires SSL certificate
Directory Listings Standardized binary format Human-readable text
Command Execution Extends SSH capabilities FTP commands only

Modern FTP Alternatives

  • Aspera: High-speed file transfer (petabytes/day)
  • Signiant: Media file transfer acceleration
  • WeTransfer: Consumer-friendly file sharing
  • Cloud Storage: S3, Google Cloud Storage with presigned URLs
  • WebDAV: Web-based distributed authoring

📤 Over 2 billion files are transferred via FTP protocols daily worldwide.
