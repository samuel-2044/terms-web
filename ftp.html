File Transfer Protocol (FTP)

A standard protocol used to transfer files between client and server systems.

Definition of FTP

File Transfer Protocol (FTP) is a network protocol and system used to transfer files between client computers and servers over a TCP/IP network connection. Standardized in 1971, FTP was one of the earliest internet protocols and remains widely used today for reliable file transfer operations despite security limitations. FTP operates using a client-server model where the client initiates connections and the server responds, using separate control and data connections to manage commands and file transfer independently. While traditional FTP operates in plain text without encryption, modern variants like SFTP (SSH File Transfer Protocol) and FTPS (FTP Secure) add encryption for security. FTP operates on port 21 for the control connection and either port 20 (active mode) or dynamic ports (passive mode) for data transfer.

How FTP Works - Detailed Process

FTP operates through a two-connection system: (1) The client establishes a control connection to the server on port 21, where the client sends commands (USER, PASS, LIST, GET, PUT, etc.); (2) The server responds to commands on this control connection but conducts actual file transfer on a separate data connection; (3) In active mode, the server initiates data connections back to the client; (4) In passive mode (more firewall-friendly), the client initiates data connections to the server; (5) File transfer happens separately from commands, allowing the control connection to remain open; (6) After transfer completion, either the control connection closes or remains open for additional commands.

FTP Components and Connection Modes

  • Control Connection: Persistent TCP connection on port 21 where commands are sent and responses received (text-based protocol using ASCII)
  • Data Connection: Separate connection for transferring actual file content; established per-file on port 20 (active) or negotiated ports (passive)
  • Active Mode FTP: Server initiates data connection to client; problematic behind firewalls blocking inbound connections
  • Passive Mode FTP: Client initiates data connection to server; preferred when behind firewalls or NAT; server provides port information on control connection
  • Transfer Modes: ASCII mode for text files preserves line endings during transfer; Binary mode transfers files exactly without conversion

FTP Commands and Operations

  • Authentication Commands: USER (username), PASS (password), QUIT (disconnect)
  • Directory Navigation: PWD (current directory), CWD (change directory), LIST (list files), NLST (name list)
  • File Operations: RETR (download/retrieve file), STOR (upload/store file), DELE (delete file), REN (rename)
  • Directory Management: MKD (make directory), RMD (remove directory)

Security Considerations and Evolution

  • Plain Text Vulnerability: Standard FTP sends credentials and data unencrypted, vulnerable to interception and sniffing
  • SFTP (SSH FTP): Uses SSH tunnel for encryption, adding security while maintaining FTP-like interface; recommended for sensitive files
  • FTPS (FTP Secure): Adds SSL/TLS encryption to standard FTP; backward compatible with explicit and implicit modes

Common FTP Uses

  • Uploading website files to web servers from development environments
  • Large file transfers where email or cloud storage would be impractical
  • Automated bulk file distribution to multiple servers
  • Data backups to remote FTP servers
  • Firmware updates and software distribution

How FTP Works

  • Control Connection: Sends commands and responses (Port 21)
  • Data Connection: Transfers files (Port 20 or dynamic ports)
  • Supports active and passive modes for connection handling

History

FTP was standardized in the early 1970s and became one of the earliest methods for transferring files over the Internet. It has since evolved into more secure versions like SFTP and FTPS.

Use Cases

  • Website file uploads and management
  • Software distribution
  • Large data backups and transfers

Security

  • Standard FTP is not encrypted
  • Credentials can be intercepted if not secured
  • Use SFTP or FTPS for secure transfers

Related Protocols

  • SFTP: Secure file transfer over SSH
  • FTPS: FTP with SSL/TLS encryption
  • TFTP: Lightweight file transfer protocol

FTP is still widely used, but secure versions are now preferred.

FTP Connection Modes

FTP uses different modes to establish data connections:

  • Active Mode: Server initiates data connection to client's port 20
    • Client sends PORT command with its IP and port
    • Server connects from port 20 to specified client port
    • Can fail with firewalls/NAT
  • Passive Mode: Client initiates both control and data connections
    • Client sends PASV command
    • Server responds with IP and random port number
    • Client connects to that port for data transfer
    • Better for firewalls and NAT

Common FTP Commands

  • USER: Specify username for authentication
  • PASS: Specify password
  • LIST: List files in current directory
  • RETR: Retrieve/download a file
  • STOR: Store/upload a file
  • DELE: Delete a file
  • MKD: Create a directory
  • PWD: Print working directory
  • QUIT: Close connection

FTP Response Codes

  • 1xx: Positive preliminary reply
  • 2xx: Positive completion (200 Command OK, 226 Transfer Complete)
  • 3xx: Positive intermediate (331 Username OK, need password)
  • 4xx: Transient negative (421 Service unavailable, 451 Local error)
  • 5xx: Permanent negative (550 File not found, 530 Not logged in)

SFTP vs FTPS Comparison

Feature SFTP FTPS
Protocol Base SSH (Secure Shell) FTP with SSL/TLS
Port 22 990 (implicit), 21 (explicit)
Firewall Friendly Yes - single port No - requires multiple ports
Platform Support Universal via SSH Requires SSL certificate
Directory Listings Standardized binary format Human-readable text
Command Execution Extends SSH capabilities FTP commands only

Modern FTP Alternatives

  • Signiant: Media file transfer acceleration
  • WeTransfer: Consumer-friendly file sharing
  • Cloud Storage: S3, Google Cloud Storage with presigned URLs
  • WebDAV: Web-based distributed authoring
